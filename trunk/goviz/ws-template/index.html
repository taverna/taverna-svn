GoViz - Gene Ontology Visualisation Service

tmo@ebi.ac.uk, 28th November 2003

This is the index page for the Gene Ontology Visualisation SOAP service. This service is built on top of Apache Axis version 1.1 final, and uses the simple .jws deployment mechanism. You should be able to view the WSDL to interact with the service by going here. It has been written to be used within the Taverna workflow framework, see taverna.sf.net for more information about that project.

If the service does not appear to be operational then you might like to check the built in Axis Test Page which can report some common configuration problems with the server this software is running on. The other point is that this server will not run on hosts external to the EBI, as it requires direct access to our Oracle installations.

Any questions about this service please email Tom Oinn (tmo@ebi.ac.uk)

Simple API Documentation

The GoViz service consists of both stateful and stateless operations. The stateful operations are used to build a server side state containing a subgraph of the Gene Ontology which may then be used to visualise predicted functions, locations etc. The stateless operations are intended as utility functions, especially in conjunction with the colouring methods in the stateful part of the service.

Stateful Service Methods

Method name Description
String createSession()
Create a new GoViz session on the server, this returns a session identifier that can be used in subsequent calls.
void destroySession(String sessionID) Remove a session from the server, identified by the session ID returned by the createSession call.
void addTerm(String sessionID, String GOID)
Add a GO term to the visualisation, updating the state of the named session.
String getDot(String sessionID)
Return the DOT text specifying the subgraph of the GO that contains all the terms that have been added to this session using addTerm() calls plus all the ancestors of such terms.
void markTerm(String sessionID, String GOID, String colour) Adds a specific colour parameter to the supplied term. If this term is not part of the state of the given session then this method does nothing.
void clearColours(String sessionID) Removes any colour information added to this session, restoring the default colouring behaviour.

Stateless Service Methods

Method name Description
String[] getAncestors(String GOID)
Return the IDs of all ancestors of the specified term.
String[] getParents(String GOID) Return the IDs of all immediate parent terms of the specified term.
String[] getChildren(String GOID)
Return the IDs of all immediate children of the specified term.

There is no provision for adding multiple terms in one operation as this is taken care of by Taverna's implicit iteration handlers; the service is written with the assumption that it is to be used from a workflow engine.

Output Format

GoViz provides output in the form of a DOT format text string. This is the native format for the tools from the GraphViz package from AT&T research, the homepage for which is here. If using Taverna, you should set an additional MIME type of 'text/x-graphviz' for any DOT format outputs and the result browser will automatically use a local installation of the dot tool to render the files, otherwise you'll just see the DOT text.

ID Format

The various operations that take a Gene Ontology identifier expect it to be of the form 'GO:0007601' (this example should be the term for 'vision'), the leading zeros must be included as must the initial 'GO:' constant. If the service is not behaving how you'd expect, you can test whether the terms in question are what you think they are by visiting the EBI's GO search engine here.

Colours

The colours supplied to the markTerm method can be anything that is a valid colour within the dot file format. For a list of colours that are defined by name you can go here.
