Enabling Taverna Support in the myGrid Enactment Engine

It is very easy to switch on myGrid Enactment Engine support for the XScufl workflow language. The process only involves copying some libraries and making some configuration changes. Note it is necessary to use Java 1.4.1 for the Enactment Engine. The complete process is described here.

  1. Install the myGrid Enactment Engine as directed by its accompanying user manual. You should install version 0.1.4.0 or later.
  2. From within the Taverna source distribution directory 'taverna1.0' execute 'ant enactor.ext.jar'. This builds the taverna extensions library.
  3. Copy 'taverna1.0/build/tavernaenactor.jar' to 'mygrid-wfenactor/lib'.
  4. Copy 'taverna1.0/devlib/servlet.jar' to 'mygrid-wfenactor/lib'.

For Standalone Enactor continue by:

  1. Edit the myGrid Enactment Engine configuration file 'mygrid-wfenactor/conf/standalone/mygrid.properties'. In particular change the value for 'mygrid.workflow.enactor.api.class' to 'uk.ac.soton.itinnovation.taverna.enactor.TavernaWorkflowEnactor'
  2. Edit the file 'mygrid-wfenactor/bin/runenactor.bat' by adding '..\lib\tavernaenactor.jar' and '..\lib\servlet.jar' to the classpath argument.

For Web Service Enactor continue by:

  1. Edit the myGrid Enactment Engine configuration file 'mygrid-wfenactor/conf/webservice/mygrid.properties'. In particular change the value for 'mygrid.workflow.enactor.api.class' to 'uk.ac.soton.itinnovation.taverna.enactor.TavernaWorkflowEnactor'
  2. Edit the file 'mygrid-wfenactor/bin/runwsclient.bat' by adding '..\lib\servlet.jar' to the classpath argument.
  3. Redeploy the myGrid Enactment Engine as directed by its manual.
  4. Copy 'taverna1.0/build/tavernaenactor.jar' to $TOMCAT_HOME/webapps/axis/WEB-INF/lib'
  5. Restart Tomcat.
  6. Off you go.

You can use the sample standalone and gui clients for the myGrid Enactment Engine. There are sample emboss tutorial workflows, input and user files in the directory 'taverna1.0/src/uk/ac/soton/itinnovation/taverna/enactor/broker/test'. You should find that this workflow executes far quicker than its WSFL alternative.
