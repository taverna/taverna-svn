About the project

The Taverna project aims to provide a language and software tools to facilitate easy use of workflow and distributed compute technology within the eScience community. As a component of the EPSRC funded myGrid project, Taverna is available freely under the terms of the LGPL.

Taverna is a collaboration between the European Bioinformatics Institute (EBI), IT Innovation and the Rosalind Franklin Centre for Genomic Research (RFCGR). Development is coordinated through the facilities generously provided by SourceForge.net and driven by the requirements of biologists from the Newcastle Centre for Life, amongst others.

For more information, or to download the language specification, object model and associated software, please select an appropriate item from the menu on the left of this page.

Quickstart + Tutorial

To get playing with Taverna as quickly as possible we suggest you download the binary version of the workbench (assuming that you are either on a win32 machine or that you are prepared to fiddle with the launch script). This will give you a chance to build, edit and run workflows using the embedded workflow enactment engine, see the workbench documentation in the menu to the left for more details.

We will provide packages for MacOS X and i86 Linux as soon as possible, but in the meantime it is possible for a relatively advanced user to download the win32 package, modifying the batch files to suit the target operating system.

Peter Li has written a tutorial on how to get started with Taverna, you can browse it here

What's new in Beta9?

Beta9 includes various user interface fixes and error handling improvements. Its main features, however, are tied into how it interacts with LSID authorities and data sources. This functionality is turned off in the binary releases as to do otherwise would require a rather more complex installation (two servers are required to support these functions).

This release was largely caused by an EPSRC pilot project meeting, if beta8 works for you then there's no huge reason to upgrade at the present time, but expect to see major enhancements from this meeting in the next release.

What's new in beta8?

This release includes new reliability enhancements. Specifically, you can now specify alternate processors, retry policies and such for any entity within the workflow. There is more information on how to make use of these features along with a simple tutorial.

New Reliability Features - Abobe PDF Format - 207kb - download here

Other new features include an enhanced results browser with configurable views over the resulting data, better process reporting and logging, the ability to batch save all workflow results to local disk on workflow completion, configurable initial population of the service selection panel (see conf/mygrid.properties), better user interface all round using native look and feel where possible and assorted other changes.

The other significant change for this and subsequent releases is an expansion of the development team, with more myGrid people actively writing code for Taverna. More information on the myGrid project can be found here.

What's new in beta7?

In this beta release we've added support for local services to perform operations such as set union and intersection, fetch from URL, list filtering based on regular expressions and similar. In addition, the implicit iteration support that was removed for the new data model introduction is back, enabling automatic iteration over data sets - wherever possible Taverna will now automatically reconcile type cardinality mismatches and hopefully behave sensibly. The level of complexity possible in workflows is significantly higher than previous releases due mainly to these new features, there's a PDF file available that shows a couple of examples of what is possible!
