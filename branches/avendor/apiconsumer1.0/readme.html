API Consumer 1.0, tmo@ebi.ac.uk, 25th Feb 2005

This tool is used to consume an API (existing Java library with source code) and produce a description of a subset of the methods within that API such that Taverna can then use these as modules within a workflow. Run the tool by putting your source files in the 'target' directory and running 'ant'. The build will fail if there are no classes to process as is normal with javadoc based tools.

You may need to move the tools.jar into your /jre/lib/ext directory for this process to work as the javadoc classes are not included by default in the system classpath.

This tool is for use with Taverna version 1.1 or higher (so versions from CVS at the moment, seeing as 1.1 hasn't actually been released yet).
