Known Issue Workarounds

If you are installing this code into Tomcat and you want to use JMX MBeans to administer the TavServ component, you must make sure that Tomcat gets the flag -Dcom.sun.management.jmxremote set on the command line to prevent confusion about which internal JMX server to use (this is a known issue between Tomcat and Spring). This can be done by setting that value in the CATALINA_OPTS environment variable on the command line, which in turn can be automated by adding it to your startup script.

The line added might look like this on Unix systems:

export CATALINA_OPTS=-Dcom.sun.management.jmxremote
