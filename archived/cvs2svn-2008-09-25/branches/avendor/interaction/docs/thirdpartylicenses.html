3rd Party Licenses

The interaction server uses a number of tools, icons and libraries from third parties in both build and runtime. These are detailed below along with their licenses as far as we are aware of them.

Build and language tools

The interaction server is writen in the Java language and is tested to work with versions 1.4.2_07 and 1.5.0_04 of the Java Runtime Environment (JRE) from Sun Microsystems. The home page for Java on Sun's website is at java.sun.com

We use the Ant build system from the Apache Group to manage the compilation, invocation of tests and other build time activities, the version used in this case is 1.6.2 and is available from ant.apache.org

Additional compile tools are the JUnit testing suite from www.junit.org which we use to perform unit testing of the core systems and as many of the plugins as feasible, and JCoverage GPL version from www.jcoverage.com for analysis of the unit test coverage (the proportion of source code exercised by the set of defined test units).

JCoverage in turn relies on the Jakarta-ORO text processing library at jakarta.apache.org/oro, the Jakarta Byte Code Engineering Library (BCEL) at jakarta.apache.org/bcel and the Java GetOpt library at freshmeat.net/projects/getopt/. These libraries are included in binary form within the build system in the 'devlib' directory alongside those containing the core JCoverage jar file. Licenses for these libraries are GNU General Public License (for GetOpt) and Apache License 2.0 for ORO and BCEL - assuming that the BCEL project has acquired this license as part of its move to Apache Jakarta. In compliance with these licenses the links above provide access to the source code of these components.

The Ant Doxygen library allows invocation of the native Doxygen tool as part of our build process if that tool is installed, the Ant task to link the two is hosted at ant-doxygen.sourceforge.net and is available under the Apache Software License (version unknown but probably 2.0). Doxygen itself is available from www.stack.nl/~dimitri/doxygen/ and can be used to augement Javadoc with collaboration diagrams etc.

The JReleaseInfo tool inserts an autogenerated class containing metadata about the build which is then accessible from the compiled libraries, the home is at jreleaseinfo.sourceforge.net/ and license is the Apache Software License.

Runtime Libraries

The JDOM library is used for XML handling, this library is fetched from the IBiblio Maven repository when the 'fetchjars' ant target is invoked; we use version 1.0 of the library which may be found at www.jdom.org. License is a modified Apache-style license, see www.jdom.org/docs/faq.html#a0030

Log4J is used to provide a simple and configurable logging infrastructure, we use version 1.2.8 which is available from logging.apache.org/log4j although we should probably upgrade to the most recent 1.2.11 version. I assume this is also under Apache license, we fetch the jar file at runtime as with JDOM and BeanShell so this probably doesn't affect us in any case.

We make use of the various Jakarta Commons libraries, in particular commons-discovery for the service provider interface mechanism available at jakarta.apache.org/commons/discovery/

Icon Set

We use the Crystal icon set within the interaction server web page UIs to provide all our colourful shiny icons, many thanks to Everaldo for providing this resource to the academic and open source community. More details are available at www.everaldo.com and www.yellowicon.com.
