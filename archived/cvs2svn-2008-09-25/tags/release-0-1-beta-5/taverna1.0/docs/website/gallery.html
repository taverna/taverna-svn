scufl workbench screenshots

Version 0.1.beta4, 18th June 2003, tmo@ebi.ac.uk

This page shows various screenshots taken from the beta4 version of the Scufl Workbench application. Click on the thumbnail images to zoom in.

Entire workbench sample session

 This is an example of a typical editing session, see the items below for details on particular components within the main window.

Scufl Model Explorer

 The model explorer allows the user to view and edit various aspects of the workflow. It presents the data using a JTree subclass, and provides for various different menu based interactions. In this example you can see the overall workflow inputs and outputs at the top, then the processors, data links and finally coordination constraints.
New data links are created by right clicking on a processor output or workflow input and selecting either a workflow output or processor input to which the data should be routed. The processor's outputs can be viewed by expanding the processor node in the tree.
Coordination constraints are similarly created, right clicking on the processor that you wish to govern with the constraint and then selecting the processor who's state should act as the condition. Currently this only allows you to express the 'block scheduled to running until other has completed' but this should be enough for most current use cases.

Service Selection Panel

 The service selection panel provides a 'palette' of processors which you can add to your workflow. Processor factories appear in a tree, right clicking on a leaf node will allow you to instantiate a new processor within the workflow with a name you enter into the summoned dialog box.
The service selection panel is initially empty, it may be populated by right clicking on the 'available services' text and selecting an appropriate option. You can add WSDL, nested workflow or Talisman services individually or entire SoapLab installations. In addition, you can select to initiate a web crawl to find XScufl and TScript files, or extract all possible scavenger modules from the currently loaded workflow.

Workflow Graph diagram

 The workflow diagram presents a view (read only) on the current workflow model. This uses the Dot tool from GraphViz (AT&T) to render the workflow as a PNG image and then display it in an internal frame within the workbench. There are various display options, in particular you can select whether to see all ports, no ports or only those which are bound to data links. This example shows a workflow with bound ports shown; services are colour coded according to type, green for WSDL, pink for Talisman and beige for Soaplab.
Another workflow diagram, this time with no ports shown. These representations may all be saved in their original 'dot' form from the file menu.
And again, this time with all ports shown. Note that this view, while informative, can be somewhat unwieldy, especially in the case of Soaplab services which can have an extremely large number of inputs and outputs.

XScufl Text View

 Largely for information or curiosity, this component shows the XML form of the Scufl language corresponding to the current workflow. It is this text that is saved when you select the 'save workflow' option from the file menu.

Running the workflow

 In order to run a workflow you first need to display the workflow input panel. This shows one input box for each workflow input present in the current workflow, allowing you to enter the initial data for the workflow run. You can see the generated XML input document if you're curious, and run the workflow by hitting the 'play' icon. This will run the workflow using an embedded instance of the workflow enactor, if you've got to this point then you have everything you need already.
When the play icon is selected, a workflow invocation dialog is displayed. This consists of three components; a table showing the status of each processor in the workflow and their start and end times, a text area containing the output document and a final text area containing the generated workflow provenance returned by the enactor.
