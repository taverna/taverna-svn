Tools available via ant

tmo@ebi.ac.uk, 13th May 2003

There are several tools for manipulating XScufl files available via ant targets. There is an example workflow that uses the three processor types, coordination blocks etc in the hybridworkflow folder in the top level of the taverna code.

Convert XScufl to Dot

ant tools.dot -Dfile=xscufl_file.xml -Doutfile=dotfile_output.dot -Dports=[all|none|bound]

Reads the XScufl file supplied as 'file', parses it into a ScuflModel and produces a Dot file which may then be rendered or otherwise processed by the dot tool from AT&T, i.e.

dot -Tpng outfile.dot > graph.png

to produce a png image from your workflow. If you have access to tools such as Adobe Illustrator you can use the -Tsvg option to produce a vector graphic form that is better suited to production of posters etc. The best output is provided when you use the 'bound' ports setting, this shows ports on the processors that are bound to workflow sources, sinks or other ports.

Graphical component demos

For the curious, the initial very much pre-alpha versions of the swing gui components can be tested using

ant tools.demo -Dfile=xscufl_file.xml

This will show two windows, one with a tree view of the workflow and one with a currently semi functional view generated by the dot program. Note that to run this you must have dot installed correctly and visible on your PATH. The components (tree view and graphic) can be run seperately by the targets tools.tree and tools.diagram respectively.
