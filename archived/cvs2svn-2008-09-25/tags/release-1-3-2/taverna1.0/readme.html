Taverna Build System

tmo@ebi.ac.uk, 23rd March 2005

Taverna is built using the Ant tool from the Apache Software Foundation. You can find it here : http://ant.apache.org, and will need at least version 1.6 to run this particular build. You will also need the Java SDK, version 1.4.2 is strongly suggested. Compiling under Java 5 may or may not work - we don't support it (and will not until all our target platforms have this version available).

Configuring the build

You may need to add an explicit 'set proxy' stage to the build file if you are behind a firewall as parts of the build require network access. Please consult the ant manual for more details on how to do this.

Default build target - 'ant'

The default target for the build system is the workbench, you must have previously called the fetchjars target to ensure you have copies of the various 3rd party jar files present (we don't distribute these in the source or cvs distributions).

Jar file gather - 'ant fetchjars'

This target, which requires http and ftp based network access from your compile machine, fetches a set of .jar files from external sources. This target must be called after a checkout from CVS or source download, and can be undone with 'ant purgejars'.

Doxygen - 'ant doxygen'

In order to be able to use Doxygen from here you will need to install both Doxygen itself and also dot.exe and its associated libraries.

  • Dot and support libraries - need to be put on your PATH somewhere.
    • http://www.research.att.com/sw/tools/graphviz/dist/dot.exe
    • http://www.research.att.com/sw/tools/graphviz/dist/thirdparty.zip
  • Doxygen, you need version 1.2.18
    • ftp://ftp.stack.nl/pub/users/dimitri/doxygen-1.2.18-setup.exe

When you have these prerequisites, running the Doxygen task will generate HTML documentation in the build/doxygen folder.
